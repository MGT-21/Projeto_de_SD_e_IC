Sobre o projeto

O projeto tem o objetivo de combater a passagem irregular em cancelas automáticas, que compromete a segurança e o controle de acesso em condomínios, estacionamentos e centros empresariais.

Por meio de duas cancelas e sensores para impedir entradas não autorizadas, visamos garantir maior segurança, rastreabilidade e eficiência no controle veicular.

Cancela Anti-Evasão Inteligente Arduino

Itens necessários

Softwares

  • Arduino IDE v2.3.6
  • Tikercard site

Componentes (core)

  • 14x jumpers macho-macho
  • 8x jumpers femea
  • 2x leds (1 verde e 1 vermelho)
  • 2x resistores (220Ω~300Ω)
  • 1x Botão
  • 2x Servos Motores
  • 2x Sensores ultrassonicos (HC-SR04)
  • 1x Placa Arduino Uno
  • 1x Protoboard

Itens opinicionais:

  • Caixa de papelão
  • Palito de picolé (cancela)
  • +6 Jumpers fêmea (colocar botão e leds fora da protoboard)

Tutorial

Replique o nosso protótipo do tinkercad em seu arduino. Fique a vontade para escolher as portas de entrada/saida que você achar melhor.

As portas de sinal estão sinalizadas como DIGITAL (PWM ~)

Guia de funcionamento

  1. O primeiro sensor idenfica se há carro ou não
  2. Identificado um carro ela espera o carro passar e a primeira cencela é fechada
  3. É assionado o botão e o led fica verde para indicar que pode prosseguir
  4. A segunda cancela abre e só se fecha quando o carro passar com segurança
  5. Após a passagem do carro a segunda cancela se fecha, a led fica vermelha e a primeira se abre

Guia de código

#include <Servo.h>

Servo servo_7;
Servo servo_6;

//Constantes para armazenar os pinos de cada componente
const int trig_6 = 10;
const int echo_6 = 9;

const int trig_7 = 2; 
const int echo_7 = 3; 
const int PINO_BOTAO = 8;

//Váriaveis para manipulação
double distancia6 = 0;
double distancia7 = 0;
bool cancelaAberta = false;

//Parte que associa as constantes aos devidos sensores
void setup()
{
  //motor direito
  servo_7.attach(7, 500, 2500);  

  //motor esquerdo
  servo_6.attach(6, 500, 2500);  

  //sensor direito
  pinMode(trig_7,OUTPUT); 
  pinMode(echo_7,INPUT); 

  //sensor esquerdo
  pinMode(trig_6,OUTPUT); 
  pinMode(echo_6,INPUT); 

  pinMode(PINO_BOTAO, INPUT_PULLUP); 
  pinMode(LED_VERDE, OUTPUT);
  pinMode(LED_VERMELHO, OUTPUT);
  Serial.begin(9600); 
}

//Parte principal do código
void loop()
{
  ligar_led_vermelho();

  //O sensor verifica se tem carro próximo
  double dist_sensor6 = verificarPresenca6();
  do {
    dist_sensor6 = verificarPresenca6();
  } while (dist_sensor6 > 7);

  //O sensor fica verificando até o carro passar completamente
  do {
    dist_sensor6 = verificarPresenca6();
  } while(dist_sensor6 < 10);

  fecharCancela6();
  bool carroDentro = true;

  //Enquanto o carro estiver dentro, espera o motorista apertar o botão para liberar o acesso
  do {
    if (digitalRead(PINO_BOTAO) == LOW) {
      abrirCancela7();

      //Sensor fica verificando até o veículo passar completamente
      double dist_sensor7 = verificarPresenca7();
      do {
        dist_sensor7 = verificarPresenca7();
      } while(dist_sensor7 < 14);

      carroDentro = false;
      fecharCancela7();
      abrirCancela6();
    }
  } while(carroDentro == true);
}

Problemas e Lições

Problemas Encontrados:

  • Identificação da placa pelo software Arduino IDE:

    Solução: Baixamos o software da Robocore. Outra alternativa seria rodar o projeto em uma distribuição Linux.

  • Identificação dos fios referentes a cada componente:

    Solução: Usamos um padrão de cor para o GND, VCC e sinal para facilitar o entendimento.

  • Posicionamento de motores e sensores na caixa:

    Solução: Trocamos a caixa que a princípio era muito pequena e usamos cola quente para fixar alguns elementos evitando má conexão e outros problemas.

Lições Aprendidas:

  • Adote padrões sempre que possível nos cabos do Arduino
  • Mantenha o código do Arduino o mais fácil de entender possível e modularizado
  • Evite redundância nos códigos
